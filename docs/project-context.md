# Project Context

## `frappe`

- Framework and admin terminology
- More technical strings, settings, permissions, metadata, automation
- Prefer clear platform language over heavy business jargon

## `erpnext`

- Finance, sales, purchasing, inventory, manufacturing, HR
- Prefer terminology common in Brazilian ERP usage
- Stay especially consistent for documents, statuses, and accounting concepts

## `crm`

- Leads, opportunities, contacts, pipeline, activities
- Keep sales funnel vocabulary consistent across the project

## `helpdesk`

- Tickets, queues, SLAs, agents, articles, support workflows
- Prefer support terminology common in Brazilian service desks
